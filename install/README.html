Sobre o Projeto

Este projeto é um Encurtador de URLs desenvolvido como desafio técnico para a empresa Topaz.

Stack tecnológica: Java 8 JAX-RS JPA + Hibernate CDI WildFly 10

1. Como Rodar o Projeto

Modo Desenvolvimento (Jetty)

mvn clean compile
mvn jetty:run

Acesse: http://localhost:8080/

Modo Produção (WildFly 10)

mvn clean package
# Copie o WAR para wildfly-10/standalone/deployments/
cp target/url-shortener.war $WILDFLY_HOME/standalone/deployments/
# Inicie o servidor
$WILDFLY_HOME/bin/standalone.sh

Acesse: http://localhost:8080/url-shortener/

2. APIs REST

MétodoEndpointDescrição
GET/Página principal
POST/api/shortenEncurtar URL
GET/{code}Redirecionar para URL original
GET/api/info/{code}Informações da URL
GET/api/healthHealth check

Exemplo de Uso

# Encurtar URL
curl -X POST http://localhost:8080/api/shorten -d "url=https://google.com&alias=google"

# Resposta
{"shortUrl":"http://localhost:8080/google","shortCode":"google","originalUrl":"https://google.com","alias":"google","accessCount":0}

# Redirect
curl -v http://localhost:8080/google
# HTTP/1.1 302 Found
# Location: https://google.com

3. Decisões de Design

Arquitetura em Camadas

O projeto segue o padrão de arquitetura em camadas:

  • Resource (JAX-RS): Endpoints REST API
  • Service: Lógica de negócio
  • Repository (JPA): Persistência CDI
  • Entity: Modelo de dados

Algoritmo Base62

Usamos Base62 para gerar shortcodes curtos:

  • Caracteres: 0-9, a-z, A-Z (62 caracteres)
  • Exemplo: ID 1000 → "sC"
  • Códigos curtos e amigáveis

Sincronização

Usamos synchronized para garantir que cada requisição seja processada de forma sincronizada, evitando códigos duplicados em cenários de concorrência.

Redirect HTTP 303

Usamos HTTP 303 (See Other) para redirect temporário. Isso indica que o recurso foi substituído pelo destino.

5. O que Faria Diferente com Mais Tempo

Melhorias Prioritárias

  • Cache Redis: Reduzir latência em reads frequentes
  • Métricas: Analytics de cliques por URL
  • QR Code: Geração automática de QR Code
  • Expiração: URLs temporárias com data de expiração

Evoluções Técnicas

  • PostgreSQL: Substituir H2 por banco relacional em produção
  • Autenticação: Controle de acesso com OAuth2
  • API Key: Limitar requisições por cliente
  • Testes de Integração: Cobertura maior com MockMvc

Trade-offs

  • Usamos synchronized em vez de fila assíncrona (simplicidade)
  • H2 em memória (desenvolvimento fácil) vs PostgreSQL (produção)
  • Redirect 303 (temporário) vs 301 (permanente) - SEO menor impacto
